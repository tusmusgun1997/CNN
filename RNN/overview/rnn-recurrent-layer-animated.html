Inside the Recurrent Layer

Watch a review flow through an RNN one word at a time — and see the output fed back into the cell at every step. 3 reviews · 5-word vocabulary · zero padding.

Oₜ = tanh(xₜ·Wᵢ + Oₜ₋₁·Wₕ) ŷ = σ(O₄·W_o) All numbers computed live

1The dataset — 3 reviews, 5 different words

Every word becomes a one-hot vector of size 5 (our whole vocabulary). A review is fed to the network one word per time step.

#ReviewTokens (after zero padding to length 4)Sentiment y
R1“movie was great” moviewasgreatPAD [0,0,0,0,0] 1 (positive)
R2“movie was boring” moviewasboringPAD [0,0,0,0,0] 0 (negative)
R3“movie was not great” moviewasnotgreat 0 (negative)
Why zero padding? The reviews have different lengths (3, 3, 4 words), but we process them in a fixed number of steps. So shorter reviews are padded with the all-zero vector [0,0,0,0,0] until every review has length 4. Watch what happens at a PAD step in the animation: xₜ·Wᵢ = [0, 0, 0] — no new information enters; only the memory keeps flowing through Wₕ.

4The weights used (shared across ALL time steps & ALL reviews)

One-hot trick: because xₜ has a single 1, xₜ·Wᵢ simply selects one row of Wᵢ — the row of the current word. These same three matrices are reused at t = 1, 2, 3, 4 for every review.
