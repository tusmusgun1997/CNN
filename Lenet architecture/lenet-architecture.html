1What is a CNN architecture?

Every CNN is an assembly line with two halves. The feature extractor (convolution + pooling, repeated) turns raw pixels into increasingly abstract patterns. The classifier (fully connected layers) turns those patterns into a decision. Data flows left → right.

2The core move: a filter scanning the image

Convolution slides a small filter (yellow window) across the input, computing one number per position. Those numbers form the feature map on the right, lighting up cell by cell. LeNet's first layer does exactly this with six 5×5 filters.

4How LeNet implements each CNN idea

C1 · C3 · C5

Convolution layers

Learnable 5×5 filters slide over the input to detect edges, strokes and shapes. Depth grows 1→6→16→120 as later filters combine simpler ones.

S2 · S4

Subsampling (pooling)

2×2 average pooling halves width & height (28→14, 10→5), giving translation tolerance and cutting computation — LeNet's "complex cell" step.

F6

Fully connected

The 120‑vector from C5 feeds 84 densely‑connected neurons that mix all features into class evidence — the reasoning stage.

Output

Classification head

10 output units (one per digit). The original used Gaussian RBF units; modern LeNet uses softmax to produce probabilities that sum to 1.

tanh

Non‑linearity

LeNet used a scaled tanh activation after each layer (today we'd often reach for ReLU) to let the stacked layers learn non‑linear patterns.

≈60k

Tiny by design

Only ~61.7k parameters thanks to weight sharing — small enough to train on 1990s hardware, yet it cracked handwritten‑digit recognition.
